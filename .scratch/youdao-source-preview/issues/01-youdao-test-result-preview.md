# 01 — 有道词典测试结果预览

**What to build:** 在翻译源设置中，用户可以以可编辑的默认单词 `example` 对有道词典执行一次真实查询。成功后，用户可以打开“查看结果”Dialog，完整检查该单词由有道返回并经扩展后台结构化后的内容；这份结果仅用于当前 Settings 会话的检查，不保存为词汇数据或设置。

**Blocked by:** None — can start immediately.

**Status:** ready-for-agent

- [ ] 测试词默认是 `example`，用户可修改并重新执行“启用并测试”；页面只提交测试词，不能指定任意 URL 或翻译源。
- [ ] 权限拒绝、连接失败、查询无结果和解析失败均在翻译源页面显示明确状态，不打开结果 Dialog。
- [ ] 成功测试后显示“查看结果”；最近一次成功结果只在当前 Settings 会话可查看，下一次成功测试会替换该结果。
- [ ] “查看结果”打开单栏、纵向滚动的 Dialog：无左侧目录、无左右分栏、无音频播放、无保存动作。
- [ ] Dialog 按有道的原始内容分类连续展示本次真实返回的结构化内容，包括可用的基础释义、词形变化、网络或专业释义、英英或双语释义、词组短语、近义词辨析和例句；本次未返回的区块不显示。
- [ ] 任何包含多个独立条目的有道区块都以结构化条目传递，不能将整个区块的文本压成一个字符串。词形变化保留名称和值并渲染为紧凑的两列列表；词组短语保留短语和释义；例句保留英文、中文和来源，并以逐条纵向列表显示。
- [ ] 设置页不接收有道原始 HTML；后台仅向其返回为预览定义的结构化结果，且请求继续受有道固定 origin 和显式权限约束。
- [ ] 自动化测试覆盖测试词输入、权限和失败状态、成功结果传递、Dialog 打开和关闭、长内容滚动，以及部分内容区块缺失时的展示。
