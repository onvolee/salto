# 05 — 模板持久化和仓库操作

**What to build:** 用户可以通过后台仓库创建、复制、删除、更新查询模板，设置和恢复默认模板，并在扩展安装或数据丢失时幂等地种子默认模板。

**Blocked by:** 03, 04（已完成）

**Status:** ready-for-agent

- [ ] 模板创建：从默认或另一个模板创建新模板，生成碰撞安全的 ID
- [ ] 模板复制：复制现有模板并保留字段定义，生成新 ID
- [ ] 模板删除：删除用户创建的模板，防止删除最后一个可用模板
- [ ] 模板更新：更新模板名称、字段定义、时间戳
- [ ] 默认模板选择：持久化一个默认模板 ID
- [ ] 默认模板恢复：当选定的默认模板被删除或损坏时，恢复到系统默认
- [ ] 幂等种子：在扩展安装或数据丢失时种子系统默认模板
- [ ] 前向迁移：从早期设置形状的幂等迁移路径
- [ ] 仓库测试：覆盖创建、复制、更新、删除、设置默认、恢复、迁移
- [ ] 时间戳：存储模板创建和更新时间戳用于迁移和故障排除
- [ ] 表单验证：使用 @tanstack/react-form + zod 处理模板编辑表单
- [ ] shadcn 组件安装：安装 `dialog`（模板删除确认）、`dropdown-menu`（模板操作菜单）
